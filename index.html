Game Engines

Unity

Unity is the engine that i have used the most, I've been using it since I was 19 years old. It was the first motor i used in High Certificate, there i developed a Survival FPS as a first project and then a 2D Turn-based combat System in a team size of 3 using GitHub in one month. And worked afterwards on two profesional projects.

C# Unity

Unreal Engine 5

Unreal is a C++ engine I learned to use through an online course and mostly on my own. The project i worked on my own was a FPS using Events, Blueprints, C++, Animations, Gameplay... I would use this engine with 3D realistic graphics or open-world games. It also has a lot of free commercial assets every month to add to your collection permanently, which is amazing!

C++ Unreal Engine 5

Godot

Godot is an Event-Oriented engine ( Signals! ) which made me love it. It's one of my favorite engines without a doubt, i really would like to work in a project with it. It can be used with C# and C++ but the GDScript was easy to learn and use with all the documentation so i will recommend that language to use.

GDScript Godot Engine
